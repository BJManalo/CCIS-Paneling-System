
# Recommendation for Archive Interface Polish

Based on your request to separate the **Archives** from the main navigation tabs, here is the proposed design improvement plan:

### 1. New Location: Top Navigation Bar
Instead of having "Archives" as a button in the bottom navigation (which is usually reserved for daily operations), we will move it to the **top header**.

*   **Placement**: Positioned next to the `CCIS Paneling System` logo in the top-left corner.
*   **Visual Style**: A sleek, circular icon button (using the `inventory_2` material icon) that matches the premium aesthetic of the admin dashboard.
*   **Benefit**: This clears up space in the bottom navigation for more frequently used tools while keeping historical data easily accessible but "separated" from current active projects.

### 2. Removal from Bottom Navigation
We will remove the "Archives" tab from the `.bottom-nav` across all admin pages:
*   `admin.html`
*   `reports.html`
*   `schedule.html`
*   `accounts.html`
*   `archive_studies.html`

### 3. Implementation Details
*   **HTML**: Add a `<button id="archiveNavBtn">` in the header next to the logo.
*   **CSS**: Style the button with a subtle hover effect and tooltip "View Archives".
*   **JavaScript**: Ensure the logout and navigation logic across these pages is updated to support this new global header button.

### Why this is better:
- **Clarification of Scope**: Archives represent the "past," while the other tabs represent the "present." Placing it in the header visually signals this difference.
- **Improved Hierarchy**: The bottom menu becomes more focused on current management tasks.
- **Modern UI**: Navigation headers with utility icons (like Archives or Notifications) are a standard in premium dashboard designs.
